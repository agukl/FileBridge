# 正规化推进清单

这份清单用于把 FileBridge 从项目形态推进到可持续交付的软件产品形态。

## 已完成的基础

- 产品名已统一为 `FileBridge`。
- 后端已作为 `FileBridgeAgent` Windows Service 运行。
- 安装目录收敛到 `C:\Program Files\FileBridge`。
- 数据目录收敛到 `C:\ProgramData\FileBridge`。
- 数据库已迁移为 SQLite，不依赖外部 MySQL。
- 安装包使用 Inno Setup，具备安装、卸载和完全清理流程。
- 构建脚本支持可选代码签名，签名内部 exe 和最终安装包。
- 发布前增加软件行为检查，覆盖 loopback API、LocalService、卸载边界、token、日志轮转和私钥 staging。
- 授权使用离线 Ed25519 公私钥许可证。
- 开发目录、工具目录、打包输出目录已拆开。

## 下一阶段建议

| 优先级 | 事项 | 目的 |
| --- | --- | --- |
| P0 | 固定发布流程和验收清单 | 每个安装包可复现、可回滚、可解释 |
| P0 | 安装包和可执行文件签名 | 降低客户机器安全软件拦截概率 |
| P0 | 授权网站只保存私钥 | 客户端永远只内置公钥 |
| P1 | 建立版本变更记录 | 方便客户升级和现场定位 |
| P1 | 增加服务健康诊断页 | 让实施人员少翻日志 |
| P1 | 增加数据库备份和恢复说明 | 降低现场升级风险 |
| P2 | 替换 `com.acme` 包名和应用标识 | 与正式公司/产品主体一致 |
| P2 | 增加自动化安装测试 | 覆盖全新安装、升级、卸载和完全清理 |
| P2 | 梳理第三方组件许可 | 为商业分发做合规准备 |

## 正式商用前必须确认

- 公司主体、软件名称、安装包名称、服务名称全部一致。
- 公钥版本号、私钥保管方式、授权网站签发流程已固化。
- 开发私钥没有进入安装包、客户包、日志或归档包。
- 客户机器断网时仍能完成安装、授权导入和本机运行。
- 普通用户能读懂安装失败提示，实施人员能根据日志定位问题。
- 卸载流程符合预期：普通卸载保留数据，完全清理删除数据。

## 版本发布节奏

项目从 `1.0.0` 开始使用三段式版本，当前版本和升版清单见 [versioning.md](versioning.md)：

```text
主版本.次版本.修订版本
```

- 主版本：安装路径、授权协议、数据库结构或重大兼容性变化。
- 次版本：新增业务能力或 UI 模块。
- 修订版本：问题修复、日志增强、安装脚本加固。

发布时同步更新：

```text
CHANGELOG.md
docs\release-process.md
package\output\FileBridge Setup x.y.z.exe
```
